\doxysubsubsubsubsection{Подробное описание}
Uninterpreted records may be attached to graphs, subgraphs, nodes, and edges for efficient operations on values such as marks, weights, counts, and pointers needed by algorithms. Application programmers define the fields of these records, but they must be declared with a common record header \doxylink{group__cgraph__rec_ga2bd56252330358f27af8321f650a1e8a}{Agrec\+\_\+t}.

A unique string ID (stored in \doxylink{struct_agrec__s_a81a026cc6c0a0759940833065f88971d}{Agrec\+\_\+s::\+name}) must be given to each record attached to the same object. Cgraph has functions to create, search for, and delete these records. The records are maintained in a circular list, with \doxylink{struct_agobj__s_ac595be4c079abaf534843935f4db97b6}{Agobj\+\_\+s::\+data} pointing somewhere in the list. The search function \doxylink{group__cgraph__rec_ga5cb2db42cf08631f32b7653114cddaee}{aggetrec} has an option to lock this pointer on a given record. The application must be written so only one such lock is outstanding at a time.

Records are created and managed by Libcgraph. A programmer must explicitly attach them to the objects in a graph, either to individual objects one at a time via \doxylink{group__cgraph__rec_ga6bdb778abe4c0e93b957329520e31917}{agbindrec}, or to all the objects of the same class in a graph via \doxylink{group__cgraph__rec_gaf60635002636d33f14f1da1c9e348f8a}{aginit}. The {\ttfamily name} argument of a record distinguishes various types of records, and is programmer defined.

Libcgraph reserves the prefix "{}\+\_\+\+AG\+\_\+"{} (in Data\+Dict\+Name, Ag\+Data\+Rec\+Name, DRName).

Internally, records \doxylink{struct_agrec__s}{Agrec\+\_\+s} are maintained in circular linked lists attached to graph objects \doxylink{struct_agobj__s}{Agobj\+\_\+s}. To allow referencing application-\/dependent data without function calls or search, Libcgraph allows setting and locking the list pointer of a graph, node, or edge on a particular record (see \doxylink{struct_agtag__s_aaa4d9a53a33a194e419ce32d9e2db3be}{Agtag\+\_\+s::\+mtflock} and \doxylink{struct_agobj__s_ac595be4c079abaf534843935f4db97b6}{Agobj\+\_\+s::\+data}). This pointer can be obtained with the macro \doxylink{group__cgraph__object_ga12a48d1c8103f642da615024eee9d7eb}{AGDATA(obj)}. A cast, generally within a macro or inline function, is usually applied to convert the list pointer to an appropriate programmer-\/defined type (eg. \doxylink{group__cgraph__graph_ga7c4edc4279776663c7e0b498d32415f5}{GD\+\_\+parent}).

To control the setting of this pointer, the {\ttfamily move\+\_\+to\+\_\+front} flag may be TRUE or FALSE. If {\ttfamily move\+\_\+to\+\_\+front} is TRUE, the record will be locked \doxylink{struct_agtag__s_aaa4d9a53a33a194e419ce32d9e2db3be}{Agtag\+\_\+s::\+mtflock} at the head of the list \doxylink{struct_agobj__s_ac595be4c079abaf534843935f4db97b6}{Agobj\+\_\+s::\+data}, so it can be accessed directly by \doxylink{group__cgraph__object_ga12a48d1c8103f642da615024eee9d7eb}{AGDATA(obj)}.

The lock protects the data pointer from being moved. Function \doxylink{group__cgraph__rec_ga5cb2db42cf08631f32b7653114cddaee}{aggetrec} reports error when data pointer and lock are reassigned.

The lock can be released or reset by a call to \doxylink{group__cgraph__rec_ga296164f98874ccee53917ebbabd08e1d}{agdelrec}.
